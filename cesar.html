1) Quais tarefas ocorrem em sequência para incrementos sala limpa?

Resposta: Planejamento de incremento; coleta dos requisitos; especificação box structure; projeto formal; verificação de correção; geração de código, inspeção e verificação; planejamento do teste estatístico; teste de uso estatístico; certificação.

2) Como funciona o método especificação box structure?

Resposta: Uma ‘caixa’ encapsula o sistema (ou algum aspecto do sistema) em algum nível de detalhe. Por meio de um processo de elaboração ou refinamento passo a passo, as caixas são refinadas em uma hierarquia em que cada caixa tem transparência referencial.

3) Qual o objetivo do teste de sala limpa?

Resposta: É validar requisitos de software demonstrando que uma amostragem estatística de casos de caso de uso foi executada de forma bem-sucedida.

4) Como funciona os métodos formais usados no desenvolvimento de sistemas de computadores?

Resposta: São técnicas de base matemática para descrever propriedades de sistemas. Esses métodos formais proporcionam uma estrutura dentro da qual as pessoas podem especificar, desenvolver e verificar sistemas de maneira sistemática, em vez de casual.

5) Qual a especificidade da caixa-preta?

Resposta: Especifica o comportamento de um sistema ou de uma parte dele. O sistema responde a estímulos específicos (eventos) aplicando um conjunto de regras de transição que mapeiam os estímulos em uma resposta.
